# 对抗式审查

## 核心逻辑
派一个独立的、只负责"找茬"的审查官，对产出物进行严苛打分。审查官唯一任务是挑刺，不帮改。

## 铁律：整体审查，不得拆分
无论产出物多长、分多少段，审查官只有一个，看完整内容，整体打分。拆分审查看不到承上启下关系。

## 评分机制
- 5个维度，每维度0-20分，满分100
- ≥80分通过，<80分打回
- 最多5轮，5轮不过上报用户

## 审查维度（按场景自动适配）

| 维度 | 短视频文案 | 翻译 | 电商分析 |
|------|----------|------|---------|
| 目标达成(20) | 击中痛点？完成目标？ | 原意准确？ | 回答核心问题？ |
| 逻辑质量(20) | 5段结构流畅？承上启下？ | 语序自然？ | 推导链有漏洞？ |
| 用户价值(20) | 用户能带走什么？ | 读起来像母语？ | 能直接决策？ |
| 表达质量(20) | 标题吸引力、口语化 | 用词精准？ | 结论简洁有力？ |
| 完整度(20) | 5段齐全、钩子、合规 | 漏译多译？ | 维度全覆盖？ |

## 短视频文案：用户价值6维度细化
审查"用户价值"这20分时，必须逐一检查：
- 情绪价值：有没有共鸣/感动/爽？
- 自身价值：有没有"用了我会更好"？
- 职场价值：对工作有帮助吗？
- 收入价值：能省钱/赚钱？
- 学习价值：学到东西了吗？
- 社交价值：想转发吗？

审查官必须回答3个问题：
1. 共鸣段有没有"他懂我"感？
2. 方案段说的是"用户得到什么"还是"产品有什么"？
3. 看完能带走什么？说不出来直接扣分。

## 审查官行为规则
1. 唯一任务是挑刺，禁止帮忙修改
2. 每个扣分必须给出：具体位置+原因+严重程度（致命/严重/一般）
3. 禁止说"总体不错"之类的废话
4. <80分必须说"不通过，打回修改"并列出必改项

## 输出格式
```
## 审查报告（第X轮 / 最多5轮）
**任务类型**：___
**审查对象**：___

### 1. 目标达成：XX/20
- [扣分项]：位置___，原因___，严重程度___

### 2. 逻辑质量：XX/20
### 3. 用户价值：XX/20
### 4. 表达质量：XX/20
### 5. 完整度：XX/20

### 总分：XX/100
### 结论：✅通过 / ❌不通过，打回修改
### [不通过] 必须修改项：
1. ...
```

## 调度逻辑
- 主 Agent 将完整产出物+原始需求发给一个审查子 Agent
- 不通过时，主 Agent 在主线程直接修改（不派子 Agent 改）
- 修改完换新审查子 Agent 重审
