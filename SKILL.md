# PDERR 工作流总调度

## 触发条件
当用户下达短视频文案、高质量翻译、电商分析、AI提示词、流程优化等高标准任务时自动启动。

## 短视频文案专属编排

### 前置层：爆款公式建模
- 加载 short-video-script Skill
- 必须先完成爆款建模表（买方思维3问 + 价值维度 + 5段骨架 + 3标题 + 钩子）
- 建模表没填完，不允许进入规划阶段

### 阶段1：规划（Plan）
- 主 Agent 自己做
- 一次只问一个问题，连续追问确认：产品/人群/平台/时长/目标/价值维度/钩子方向
- 用户确认"可以"才进入拆解

### 阶段2：拆解（Decompose）
- 主 Agent 自己做
- 基于建模表拆成逐秒脚本框架（钩子段/共鸣段/方案段/证明段/行动段）
- 确认标题3选1、封面文字、评论区置顶
- 用户确认"开干"才进入执行

### 阶段3：执行（Execute）
- 主 Agent 自己写，不派子 Agent（子 Agent 超时率高且可能写错产品）
- 加载 short-video-script Skill，按写作金标准产出完整脚本
- 产出物包含：标题+封面+逐秒口播+字幕+评论区置顶+发布正文
- 写完不直接交付，转入审查

### 阶段4：审查（Review）
- 派一个独立审查子 Agent，加载 adversarial-review Skill
- 铁律：一个审查官看完整内容整体打分，不得拆分
- ≥80分通过，<80分打回主 Agent 在主线程修改（不派子 Agent 修改）
- 修改完换新审查子 Agent 重审，最多5轮
- 5轮不过上报用户

### 阶段5：复盘（Retrospective）
- 主 Agent 自己做，加载 retrospective Skill
- 记录审查轮数、得分轨迹、扣分重灾区、经验教训
- 每次任务必须复盘，无例外

### 全局铁律
1. 执行和修改由主 Agent 做，只有审查用独立子 Agent
2. 审查必须整体进行，连贯产出物不得拆分
3. 审查打回后主 Agent 主线程修改
4. 每轮审查换新子 Agent
5. 无复盘不结束
