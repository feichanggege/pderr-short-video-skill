# PDERR 短视频文案工作流

> 适用：抖音 / 视频号 / 小红书 / 淘宝 等全平台

## 安装方式

```bash
# clawdhub 安装（推荐）
npx clawdhub install pderr-short-video-skill

# 或克隆到本地
git clone https://github.com/feichanggege/pderr-short-video-skill.git ~/.workbuddy/skills/pderr-short-video-skill
```

## 工作流六阶段

当用户要求"写短视频文案"时，**无条件启动 PDERR 工作流**，不许跳步。

---

## 前置层：爆款建模（必须完成）

动笔前必须填完以下建模表，**没填完不准进入规划阶段**：

```
【爆款建模表】
== 买方思维3问 ==
1. 用户刷视频是为了什么？答：___
2. 这条视频能给他什么？答：___
3. 看完后生活有什么变化？答：___

== 价值维度 ==
主攻（选1-2个）：情绪/自身/职场/收入/学习/社交
辅助（选0-1个）：___

== 爆款结构5段 ==
钩子句（前3秒）：___
共鸣句（3-15秒）：___
方案句（15-40秒）：___
证明句（40-55秒）：___
行动指令（最后5秒）：___

== 标题（3个备选，注明公式）==
== 流量钩子 ==
```

---

## 阶段1：规划（Plan）

主 Agent 确认以下信息（一次只问一个问题，用户确认"可以"才进拆解）：
- 产品/内容主题
- 目标平台（抖音/视频号/小红书/淘宝/拼多多）
- 目标人群（年龄/性别/职业）
- 视频时长（15s/30s/60s）
- 核心目标（转化/种草/品牌/涨粉）
- 产品核心卖点（≥3个）

---

## 阶段2：拆解（Decompose）

基于建模表拆成逐秒框架：

| 时段 | 作用 | 失败后果 |
|------|------|---------|
| 前3秒（钩子） | 让人停下来 | 直接划走 |
| 3-15秒（共鸣） | 让人感受价值 | 觉得没关系划走 |
| 15-40秒（方案） | 给用户带走点什么 | 没收获划走 |
| 40-55秒（证明） | 让人信 | 觉得是广告划走 |
| 最后5秒（行动） | 让人动 | 互动率低不推流 |

用户确认"开干"才进入执行。

---

## 阶段3：执行（Execute）

**主 Agent 自己写，不派子 Agent。**

产出物（缺一不可）：
1. **标题**（带话题标签，格式：`标题：XXXX\n#话题1 #话题2`）
2. **封面文字**（≤10字，吸引眼球）
3. **口播正文**（60秒节奏，口语化）
4. **发布正文**（≤300字，视频号正文描述）

铁律：
- 方案段说"用户得到什么"，不是"产品有什么"
- 每10秒一个完播悬念点
- 合规：无违规词、无功效词
- 口语化，不许书面语

---

## 阶段4：审查（Review）

派独立审查子 Agent，5维度整体打分：

| 维度 | 分值 | 评分要点 |
|------|------|---------|
| 目标达成 | 20 | 击中痛点？完成目标？ |
| 逻辑质量 | 20 | 5段结构流畅？承上启下？ |
| 用户价值 | 20 | 6维度：情绪/自身/职场/收入/学习/社交 |
| 表达质量 | 20 | 标题吸引力、口语化、合规 |
| 完整度 | 20 | 5段齐全、钩子、行动指令 |

- **≥80分通过，<80分打回主 Agent 修改**（主线程修改，不派子 Agent）
- 修改后换新审查官重审
- 最多5轮，5轮不过上报用户

---

## 阶段5：复盘（Retrospective）

记录：
- 审查轮数 + 得分轨迹
- 主要扣分维度（标注P0/P1/P2）
- 本次经验教训
- 规律性问题（出现3次以上 → 建议更新 Skill）

---

## 全局铁律

1. 执行和修改由主 Agent 做，只有审查用独立子 Agent
2. 审查必须整体进行，连贯产出物不得拆分
3. 审查打回后主 Agent 主线程修改
4. 每轮审查换新子 Agent
5. 无复盘不结束
6. 建模表没填完不准写文案
