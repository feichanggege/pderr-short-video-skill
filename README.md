# PDERR 短视频文案工作流 Skill

> 短视频爆款文案创作 SOP，包含**建模 → 规划 → 拆解 → 执行 → 审查 → 复盘**六阶段闭环，适用于抖音、视频号、小红书、淘宝等全平台。

```bash

cp -r retrospective ~/.workbuddy/skills/
cp -r SKILL.md ~/.workbuddy/skills/pderr-orchestrator/SKILL.md
```

将 Skill 文件路径写入项目配置或直接用提示词调用：

```markdown
你是一个短视频文案专家。每次创作短视频文案时，严格按以下PDERR工作流执行：
```

## PDERR 六阶段详解

### 前置层：爆款建模（必须完成）

```
【爆款建模表】
== 买方思维3问 ==
1. 用户刷视频是为了什么？答：___
2. 这条视频能给他什么？答：___
3. 看完后生活有什么变化？答：___

== 价值维度 ==
主攻（选1-2个）：情绪/自身/职场/收入/学习/社交

== 爆款结构5段 ==
钩子句（前3秒）：___
共鸣句（3-15秒）：___
方案句（15-40秒）：___
证明句（40-55秒）：___
行动指令（最后5秒）：___

== 标题（3个备选）==
== 流量钩子 ==
```

**铁律：建模表没填完，不准进入规划阶段。**

---

### 阶段1：规划（Plan）

确认基础信息：
- 产品/内容主题
- 目标平台（抖音/视频号/小红书/淘宝/拼多多）
- 目标人群（年龄/性别/职业/兴趣）
- 视频时长（15s/30s/60s/3分钟）
- 核心目标（带货转化/种草/品牌曝光/涨粉）
- 产品卖点（≥3个）

---

### 阶段2：拆解（Decompose）

基于建模表拆成逐秒脚本框架：

| 时段 | 作用 | 买方视角 |
|------|------|---------|
| 前3秒 | 钩子，让人停下来 | "这和我有关吗？" |
| 3-15秒 | 共鸣，让人感受到价值 | "他懂我，继续看" |
| 15-40秒 | 方案，给用户带走点什么 | "这对我有用吗？" |
| 40-55秒 | 证明，让人信 | "是真的吗？" |
| 最后5秒 | 行动，让人动 | "我做什么？" |

---

### 阶段3：执行（Execute）

**主 Agent 自己做，不派子 Agent。**

产出物清单：
- 标题（含话题标签）
- 封面文字（≤10字）
- 逐秒口播文案
- 字幕文案
- 评论区置顶（可选）
- 发布正文描述（可选）

---

### 阶段4：审查（Review）

独立审查官，5维度打分：

| 维度 | 短视频文案评分要点 |
|------|-----------------|
| 目标达成(20) | 击中痛点？完成目标？ |
| 逻辑质量(20) | 5段结构流畅？承上启下？ |
| 用户价值(20) | 6维度检查：情绪/自身/职场/收入/学习/社交 |
| 表达质量(20) | 标题吸引力、口语化、合规 |
| 完整度(20) | 5段齐全、钩子、行动指令 |

- **≥80分通过，<80分打回修改**
- 最多5轮，5轮不过上报用户
- 每次修改后换新审查官重审

---

### 阶段5：复盘（Retrospective）

记录：
- 审查轮数、得分轨迹
- 主要扣分维度
- 经验教训
- 规律性问题（出现3次以上 → 建议优化 Skill）

## 6大用户价值维度

| 价值类型 | 用户内心OS | 适用场景 |
|---------|-----------|---------|
| 情绪价值 | "看完心情好了/爽了/感动了" | 种草、品牌故事 |
| 自身价值 | "让我显得更好/更健康" | 美妆、穿搭、食品 |
| 职场价值 | "让我工作更厉害" | 技能、工具 |
| 收入价值 | "让我省钱/赚钱" | 带货、选品 |
| 学习价值 | "让我长见识了" | 知识、测评 |
| 社交价值 | "我要转发/显得懂行" | 热点、反常识 |

---

## 合规要求

严格遵守中国广告法：
- ❌ 禁止：最、第一、顶级、独家、100%有效、无效退款
- ✅ 允许：具体数据（"3分钟上手"）、真实案例、用户视角描述
