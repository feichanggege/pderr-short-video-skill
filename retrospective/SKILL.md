# 复盘迭代

## 触发条件
每次任务（经过规划→拆解→执行→审查）完成后必须触发，无例外。

## 复盘数据采集
1. 任务概要：做了什么、什么场景
2. 审查数据：几轮通过、每轮得分、哪些维度被扣分最多
3. 效率数据：一次通过率、平均打回轮数
4. 关键发现：什么做得好可复用、什么做得差要避免、有没有规律性问题

## 经验分级写入
| 经验类型 | 写入位置 |
|---------|---------|
| 当次细节 | diary/YYYY-MM-DD.md |
| 规律性教训 | MEMORY.md |
| Skill优化建议 | 提议给用户确认 |

## 写入格式
```
### YYYY-MM-DD HH:MM - [复盘] <任务主题>
- **场景**：___
- **审查轮数**：X轮
- **得分轨迹**：第1轮XX分 → ... → 最终XX分
- **主要扣分维度**：___
- **关键教训**：___
- **可复用经验**：___
- **改进建议**：___
```

## 自我进化机制
1. 每次复盘先读已有教训，避免重复记录
2. 同类问题出现3次以上 → 标记为"系统性问题"，建议修改 Skill
3. 正面经验连续5次有效 → 固化到 Skill 核心指令中
